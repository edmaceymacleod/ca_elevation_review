# CI/dev constraints lock for the engine — DO NOT hand-edit version numbers.
#
# Why this exists: pyproject.toml deliberately declares dependencies as `>=`
# floors so ca-elevation-engine stays installable alongside other packages.
# That flexibility makes CI non-deterministic — a fresh upstream release (e.g. a
# new numpy whose bundled stubs break mypy) can turn CI red with zero code
# change. This file pins the exact resolved set CI installs, so runs are
# reproducible, WITHOUT constraining what downstream users get from pyproject.
#
# Used as a pip constraints file (does not itself install anything):
#   pip install -c engine/requirements-dev.txt ruff mypy pytest pytest-cov "./engine[report]"
#
# IMPORTANT: built on the OLDEST Python in the CI matrix (3.10). A flat pin set
# must install on every matrix Python; building on the floor guarantees that
# (e.g. numpy 2.2.6 spans 3.10–3.13, whereas numpy>=2.3 drops 3.10). Verified
# to install + pass ruff/mypy/pytest on 3.10, 3.11, and 3.12. The py<3.11
# backports (tomli, exceptiongroup) are simply ignored by pip on 3.11+.
#
# Bumped by Dependabot (weekly, see .github/dependabot.yml) in a reviewable PR.
# To regenerate: python3.10 -m venv .v && .v/bin/pip install "./engine[report]"
# ruff mypy pytest pytest-cov && .v/bin/pip freeze, then drop the self-reference.
